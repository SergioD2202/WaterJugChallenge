Water Jug Problem
What is the Water Jug Problem
Build an application that solves the Water Jug Riddle for dynamic inputs (X, Y, Z). The simulation should have a UI (if SPA) to display state changes for each state for each jug (Empty, Full or Partially Full). You have an X-gallon and a Y-gallon jug that you can fill from a lake. (Assume lake has unlimited amount of water.) By using only an X-gallon and Y-gallon jug (no third jug), measure Z gallons of water.
How to solve the Water Jug Problem
In order to solve this problem it was implemented a BFS algorithm (Breadth-First Search), the different decisions taken were considered as nodes of a tree, then if the option of the operation was in a range between 0 and the whole capacity of the buckets and it was not visited yet then the node was considered in the queue, the algorithm travels around all the posibilities at the same time until it founds a solution (represented as a node), once the node is found the application backtracks the father of the node and the father of that father until it reaches the root node, reverses the order and finally delivers the answer.
